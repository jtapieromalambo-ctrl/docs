This quickstart gets a Signia development server running against SQLite, with the bundled `ffmpeg` binaries and the minimum set of environment variables. For a full walkthrough — PostgreSQL, OAuth, email, and production gotchas — see [Local installation](/installation).

## Prerequisites

Before you begin, you must have:

- **Python 3.11** with `pip` 23 or newer.
- **Git**.
- A **Groq API key** from [console.groq.com](https://console.groq.com) for the LSC grammar layer.

`ffmpeg` is required for audio transcription, but the repository ships native binaries under `ffmpeg/` and the translator adds that directory to `PATH` at runtime. You only need to install `ffmpeg` system-wide if you remove that folder.

## Get started

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone <repository-url>
    cd SIGNIA
    ```
  </Step>
  <Step title="Create and activate a virtual environment">
    **Linux / macOS:**

    ```bash
    python3.11 -m venv ENT
    source ENT/bin/activate
    ```

    **Windows (PowerShell):**

    ```powershell
    python -m venv ENT
    .\ENT\Scripts\Activate.ps1
    ```
  </Step>
  <Step title="Install dependencies">
    ```bash
    pip install -r requirements.txt
    ```
  </Step>
  <Step title="Create a minimal .env file">
    Create `.env` next to `manage.py` with the smallest set of variables needed to boot:

    ```env
    SECRET_KEY=replace-with-a-long-random-string
    DEBUG=True
    GROQ_API_KEY=your-groq-api-key
    ```

    Omit `DATABASE_URL` to fall back to the bundled `db.sqlite3` automatically.

    Generate a `SECRET_KEY` with:

    ```bash
    python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
    ```
  </Step>
  <Step title="Apply migrations and run the server">
    ```bash
    python manage.py migrate
    python manage.py runserver
    ```

    Visit [http://127.0.0.1:8000](http://127.0.0.1:8000).
  </Step>
</Steps>

## Next steps

- Create a superuser with `python manage.py createsuperuser` so you can access the video admin at `/admin-videos/` and train the recognition model.
- Read [Local installation](/installation) to configure PostgreSQL, email (Brevo), OAuth, and `collectstatic` for a production-like setup.
